# 키오스크 접속 문제 해결 가이드

## 문제: 키오스크 링크 접속 시 페이지가 뜨지 않음

예시 링크: `https://127.0.0.1:8084/kiosk/F5F7-C90C-6C54`

---

## 해결 방법

### ✅ 방법 1: 브라우저에서 인증서 수락하기 (추천)

#### Chrome/Edge 브라우저:
1. 키오스크 링크 접속
2. "연결이 비공개로 설정되지 않음" 또는 "사이트에 연결할 수 없음" 오류 표시
3. **"고급"** 버튼 클릭
4. **"127.0.0.1(안전하지 않음)로 이동"** 클릭
5. 키오스크 화면이 표시됩니다! ✨

#### Firefox:
1. 키오스크 링크 접속
2. "경고: 잠재적인 보안 위험 경고" 표시
3. **"고급"** 클릭
4. **"위험을 감수하고 계속"** 클릭
5. 키오스크 화면이 표시됩니다! ✨

---

### ✅ 방법 2: 먼저 홈페이지에 접속해서 인증서 수락

1. 먼저 홈페이지 접속: `https://127.0.0.1:8084/`
2. 위의 방법대로 인증서 수락
3. 그 다음 키오스크 링크 접속: `https://127.0.0.1:8084/kiosk/F5F7-C90C-6C54`
4. 이제 바로 접속됩니다!

---

### ✅ 방법 3: 키오스크 코드가 올바른지 확인

1. 보호자 계정으로 로그인
2. 노약자 관리 > 노약자 상세보기
3. "키오스크 접속 정보" 섹션에서 올바른 코드 확인
4. 코드 형식: `XXXX-XXXX-XXXX` (예: F5F7-C90C-6C54)

---

### ✅ 방법 4: 서버가 실행 중인지 확인

```bash
# PowerShell에서 실행
cd C:\Needy_Care_System\user
.\gradlew bootRun
```

서버 로그에서 다음 메시지 확인:
```
Tomcat started on port(s): 8084 (https)
Started UserApplication in X.XXX seconds
```

---

### ✅ 방법 5: 데이터베이스에 키오스크 코드가 있는지 확인

MySQL에 접속해서 확인:
```sql
USE Needy_Care_System;

-- 키오스크 코드 확인
SELECT rec_id, rec_name, rec_kiosk_code 
FROM Care_Recipient 
WHERE rec_kiosk_code = 'F5F7-C90C-6C54';
```

결과가 없다면:
- 노약자를 다시 등록하거나
- 기존 노약자 정보를 수정해서 키오스크 코드 생성

---

## 실전 배포 시 (운영 서버)

실제 서버에 배포할 때는 **정식 SSL 인증서**를 사용하세요:
- Let's Encrypt (무료)
- AWS Certificate Manager
- Cloudflare SSL

이렇게 하면 브라우저 경고 없이 바로 접속 가능합니다!

---

## 모바일에서 접속 시

### Android:
1. Chrome으로 링크 열기
2. "사이트가 안전하지 않습니다" 경고
3. "상세정보" → "이 사이트 방문" 터치

### iPhone:
1. Safari로 링크 열기
2. "이 연결은 비공개 연결이 아닙니다" 경고
3. "상세 정보 보기" → "웹 사이트 방문" 터치

---

## 빠른 테스트

터미널에서 서버 응답 확인:
```bash
curl -k https://127.0.0.1:8084/kiosk/F5F7-C90C-6C54
```

`-k` 옵션은 인증서 검증을 건너뜁니다.

---

## 여전히 안 된다면?

1. 브라우저 개발자 도구(F12) 열기
2. Console 탭에서 에러 메시지 확인
3. Network 탭에서 요청이 가는지 확인
4. 서버 로그 확인 (IntelliJ/Eclipse 콘솔)

에러 로그를 공유해주시면 더 정확한 해결책을 드릴 수 있습니다!
